# 前端目录与后端一致（不做映射）— 手动重命名步骤

代码已改为使用与后端一致的路径（menu、dept、dict、operlog、logininfor），**需要你本地手动重命名文件夹**（因 dev 运行时可能占用导致无法自动重命名）。

## 操作前

1. **先停止开发服务**：结束 `pnpm run dev`（或 npm run dev）。
2. 在资源管理器或 IDE 中打开：`apps/web-antd/src/views/system/`

## 需要做的重命名

在 `system` 目录下：

| 当前文件夹 | 重命名为 | 说明 |
|-----------|----------|------|
| `management` | `menu` | 菜单管理 |
| `departmentManager` | `dept` | 部门管理 |
| `dictionaryManagement` | `dict` | 若已是 `dict` 则跳过 |
| `LogManager/operLog` | 整体移到 `system` 下并改名为 `operlog` | 即得到 `system/operlog/` |
| `LogManager/loginLog` | 整体移到 `system` 下并改名为 `logininfor` | 即得到 `system/logininfor/` |

最后：若 `LogManager` 已空，删除 `LogManager` 文件夹。

## 操作后

重新执行 `pnpm run dev`，路由与菜单会按后端返回的 component 路径直接解析，不再做映射。
